# ToDo ステータスを廃止したい

## 背景

ToDo アプリケーションの初期設計で、各 ToDo にはステータスを持たせるという要件を設けており、これを実装した。
ステータスには priority という number 型のフィールドがあり、これが `0` のものを `Done(完了)` とみなす仕様だった。

しかし、実際に開発を進める中で、ToDo にステータスを持たせる必要はないという結論に至った。

## 要件

1. (backend) ToDo テーブルに存在する todoStatusId カラムを削除する
2. (backend) ToDo Status テーブルを削除する
3. (backend) ToDo の CRUD API からステータスに関する処理を削除する
   - API の実装は下記
     - (routing) `backend/src/routes/todoRoutes.ts`
     - (handler) `backend/src/handlers/todoHandlers.ts`
     - (service) `backend/src/services/todo/todoServices.ts`
   - ToDo の CRUD API に対するテストも修正する
     - `backend/tests/handlers/todoHandlers.int.test.ts`
     - `backend/tests/services/todo/todoServices.test.ts`
4. (backend) ToDo ステータスの CRUD API を削除する
   - API の実装は下記
     - (routing) `backend/src/routes/todoStatusRoutes.ts`
     - (handler) `backend/src/handlers/todoStatusHandlers.ts`
     - (service) `backend/src/services/todoStatus/todoStatusServices.ts`
   - ToDo ステータスの CRUD API に対するテストも削除する
     - `backend/tests/handlers/todoStatusHandlers.int.test.ts`
     - `backend/tests/services/todoStatus/todoStatusServices.test.ts`
5. (frontend) ヘッダーのハンバーガーアイコンからステータス一覧へのリンクを削除する
6. (frontend) ToDo 一覧画面のモーダルから statusId をセットする入力枠を削除する
   - `frontend/app/api/todoApi.ts` に各種 API の呼び出しがあるがそちらもステータスに関する実装を削除する
7. (frontend) ToDo ステータス画面を削除する
   - `frontend/app/routes/todoStatus.tsx`
   - `frontend/app/components/TodoStatusTable.tsx`
   - `frontend/app/api/todoStatusApi.ts` も削除
   - `frontend/app/types.ts` から ToDo ステータス関連の型定義を削除

## 作業における注意点

- テーブルに対する変更はマイグレーションファイルを作成して行うこと
  - マイグレーションは `backend/` にある `prisma` を使うこと
    - `prisma migrate dev` && `pnpm exec prisma generate`
- 要件の番号を振った段落ごとにコミットを作成すること
